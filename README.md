StarGates is an **agentic AI incident-triage proof-of-concept** built with **IBM watsonx Orchestrate**.  
It transforms unstructured production alerts into **prioritized actions, validated hypotheses, and ITSM-ready incident tickets** by orchestrating multiple specialized AI agents.

## ğŸš€ Project Overview

Enterprise operations teams spend significant time manually triaging incidents:

- Interpreting unstructured alerts  
- Searching runbooks  
- Collecting logs and metrics  
- Creating and updating incident tickets  

This manual process slows response time, increases operational load, and leads to inconsistent decisions.

StarGates automates this workflow using a **multi-agent architecture orchestrated by IBM watsonx Orchestrate**.

Given a single alert, the system:

1. Determines severity and business impact  
2. Selects the most relevant operational runbook  
3. Collects supporting evidence (logs, metrics, signals)  
4. Generates plausible root-cause hypotheses with validation steps  
5. Produces a prioritized action checklist and a draft incident ticket  

This demonstrates how **agentic AI** can be applied to real enterprise workflows beyond conversational assistants.

## ğŸ—ºï¸ Architecture Overview

Flow:

1. An unstructured alert enters the system.  
2. The Sentinel Agent evaluates impact and selects a runbook.  
3. Evidence collection is delegated to the Evidence Collector Agent.  
4. Root-cause hypotheses are generated by the RCA Agent.  
5. The Sentinel compiles prioritized actions and an ITSM-ready incident ticket.

This design enables **selective delegation**:  
simple incidents resolve quickly, while complex incidents trigger deeper analysis.
